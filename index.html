How to Play

Play vs Computer (AI)

  1. Set your username in the Player Profile setup card at the top.
  2. Select your desired difficulty level under Difficulty Level (Easy, Medium, Hard, or Expert).
  3. Click Start Match to launch the game directly against the local engine.

Play vs Friend (Peer-to-Peer)

  1. Enter your username in the Player Profile setup card.
  2. To Host: Click Host New Room to generate a 6-character room code. Share this code, the QR invite, or the direct link with your friend.
  3. To Join: Paste your friend's room code in the input box and click Join.
  4. WebRTC establishes a direct, secure connection between both browsers instantly.

Architectural Deep-Dive

This system utilizes WebRTC (Web Real-Time Communication) to establish direct, low-latency datachannels between players. Under ordinary circumstances, browser applications require a centralized server to relay state changes. P2P Chess replaces the server entirely once the match begins.

P2P Platform Architecture

1. WebRTC Protocol Setup

WebRTC (Web Real-Time Communication) provides web browsers with direct Application Programming Interfaces (APIs) for Peer-to-Peer data exchange. To connect, players need to trade network configurations via Signaling.

2. What are STUN, TURN, and ICE?

  • ICE (Interactive Connectivity Establishment): A framework used to discover the best network paths between peers.
  • STUN (Session Traversal Utilities for NAT): Allows local clients to discover their public-facing IP and port. Most connections succeed using STUN. We use Google's public STUN servers for this discovery.
  • TURN (Traversal Using Relays around NAT): If both clients are behind restrictive firewalls (such as symmetric enterprise NATs), they cannot connect directly. In a commercial environment, TURN servers act as data relays. Because TURN servers require high bandwidth hosting, they are omitted in this free platform, meaning connections might fail on extremely strict enterprise networks.

3. Dynamic Room-Code Matchmaking

To avoid requiring complex databases, the system converts letters and numbers into a 6-character room code (e.g. A1-N2-B2). The Host spawns a PeerJS connection registering this ID. The Guest queries this exact Peer ID. Once the SDP handshake establishes the direct WebRTC channel, signaling is closed.

4. Dual-State Move Validation

To ensure game security, move validation runs on both browsers. When you execute a move, it is checked locally, sent over the WebRTC datachannel, and then re-validated by your opponent's engine before updating their screen. If a desync occurs, the engine re-synchronizes the board using standard FEN hashes.
